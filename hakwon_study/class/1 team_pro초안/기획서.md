## 기술 스택 상세

### 백엔드
- **Flask**: 웹 프레임워크
- **Werkzeug**: 비밀번호 해싱, 세션 관리
- **PyMySQL**: MariaDB 연결

### AI/ML
- **Google Generative AI (Gemini)**: 감정 분석 API

### 프론트엔드
- **Jinja2**: 템플릿 엔진
- **HTML5/CSS3**: 반응형 웹 디자인

### 데이터베이스
- **MariaDB**: 관계형 데이터베이스
  - `users` 테이블: 사용자 정보
  - `diaries` 테이블: 일기 내용, 감정, 분석 결과

### 배포 및 버전 관리
- **Git**: 버전 관리
- **Cloudtype**: 클라우드 배포 플랫폼
- **Gunicorn**: WSGI 서버

## 데이터베이스 스키마

### users 테이블
- `id`: INT (Primary Key, Auto Increment)
- `username`: VARCHAR(50) (Unique, Not Null)
- `password`: VARCHAR(255) (Not Null, Hashed)
- `created_at`: TIMESTAMP (Default: CURRENT_TIMESTAMP)

### diaries 테이블
- `id`: INT (Primary Key, Auto Increment)
- `user_id`: INT (Foreign Key → users.id)
- `content`: TEXT (일기 내용)
- `emotion`: VARCHAR(20) (행복, 보통, 우울, 분노)
- `emotion_score`: INT (0-3, 감정 점수)
- `diary_date`: DATE (일기 작성 날짜)
- `analysis`: TEXT (JSON 형식, AI 분석 결과)
- `created_at`: TIMESTAMP (Default: CURRENT_TIMESTAMP)

## 주요 기능 흐름도

1. **회원가입/로그인**
   - 사용자 입력 → 비밀번호 해싱 → DB 저장/검증 → 세션 생성

2. **일기 작성 및 분석**
   - 일기 입력 → Flask 라우팅 → EmotionAnalyzer 호출 → Gemini API 요청 → 감정 분류 → DB 저장 → 결과 페이지 표시

3. **일기 목록 조회**
   - 로그인 확인 → DB 조회 → JSON 파싱 → 템플릿 렌더링

## 보안 고려사항

1. **비밀번호 보안**: Werkzeug의 `generate_password_hash` 사용
2. **세션 관리**: Flask 세션을 통한 사용자 인증
3. **입력 검증**: 폼 데이터 유효성 검사
4. **SQL 인젝션 방지**: PyMySQL의 파라미터화된 쿼리 사용
5. **API 키 보안**: 환경 변수를 통한 API 키 관리
